Hello Windows Insiders, and greetings from the Windows Defender team! As all may have seen, starting with Windows 10 Insider Preview Build 10571 we added built in context menu (Right Click) scan for Windows Defender. Yes, that is right, it is back!!! The most missed Windows 8 feature has returned by popular demand to Windows 10 platform because users like you gave and upvoted feedback using the Feedback App.

Some example of feedback we heard:

Add back the ‘Scan with Defender..’ command to right-click menu for files/folders in File Explorer. This option should never have been removed with the switch from MS Security Essentials to Defender in Windows 8.

“It would be nice if users could choose to right click and select a scan of individual files and folders directly from that menu!”

In this article we wanted to spend some time talking about the feature work we did to enable content menu scan options and how you can perform this scan.

Context menu scan overview

This feature allows you to specify which files and folder to scan. This is really helpful when you want to determine whether a particular file or folder is malicious or not, but not find and open the UI.

This option has advantages over the custom scan exposed via Windows Defender UI. It lets you pick files in the parent folder without scanning subsequent folders.

You can use this option to scan network or external devices manually, if needed.

With the release of Windows 10 November Update, we have gotten positive feedback, and we thank you for it. We also want you to continue providing feedback via the Feedback App to help drive changes that are important to you.

Performing a scan with right click context menu scan

1. Select a file(s)/folder(s)

2. Right click mouse option on the selection

3. From the context menu scan select “Scan with Windows Defender”

4. Wait for scan to finish and see results in UI

5. If result detects any malicious file(s) then follow the steps in Windows Defender UI and clean the system

Details

1. This feature does honor Windows Defender exclusions which means it is not going to scan objects marked for exclusion.

2. For context menu option will only be available only when Windows Defender is enabled and it’s UI is not locked.

Note: if you have 3rd party software which adds context menu scan for Windows Defender, then please uninstall them manually. This feature will not uninstall your 3rd party extensions.

Having two context menu scans may cause unwanted behavior for Windows Defender scans.

Expected Behavior

Some time when scanning single office files it might say more than 1 items scanned. This is because this scans ends up scanning macro, java scripts, embedded objects in the office file

Same is expected for container files (like rar, zip etc .). This scan will do a deep scanning on container files.
